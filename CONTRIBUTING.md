# Contributing
Thank you for your interest in contributing to FNF Chart Generator Redux. I welcome bug reports and pull requests.

## Bug reports
When you report a bug you’ve encountered, make sure you keep the following points in mind when reporting it in the issues section
- **A clear explanation:** Please describe the bug you found in detail, including how it occurred and what caused it. Do not provide vague explanations.
- **Provide screenshots:** Please provide a screenshot at least one showing the bug you found so I can understand what you mean.
- **Tell your device:** Please describe the device you're using in detail so I can understand.

## Pull requests
When you fork a repository, make changes, and create a pull request, make sure you understand the following points when creating a pull request in the pull requests section
- **A clear explanation:** Just like before, you should explain it in detail, but in the pull request, describe the changes you made and explain the purpose of those changes.
- **Provide medias:** Just like before, please provide at least one piece of media whether an image or a video that shows the changes you’ve made so I can understand.
- **Check off the to-do list:** Check off the items on the to-do list to make sure you’ve tested the app and verified that there are no bugs after testing it.

## Style guide
When you make changes to the application and save the file after making those changes, you must format the style by typing the following:

Windows
```cmd
black src\Main.py
```
Linux/macOS
```bash
black src/Main.py
```
